Request编程

Python中常用的网络爬虫库之一是requests库，它提供了简洁而强大的接口来发送HTTP请求和处理响应。以下是requests库的一些常用函数接口和一个简单的使用案例介绍：

函数接口：

  1. requests.get(url, params=None, **kwargs)

    发送GET请求并获取响应。

    • url：请求的URL地址。
    • params：可选参数，字典或字节序列，作为URL的查询字符串参数。
    • *kwargs：可选参数，用于传递其他参数，如请求头(headers)、超时(timeout)等。

  2. requests.post(url, data=None, json=None, **kwargs)

    发送POST请求并获取响应。

    • url：请求的URL地址。
    • data：可选参数，字典、字节序列或文件对象，作为请求的正文内容。
    • json：可选参数，将Python对象自动编码为JSON格式并作为请求的正文内容。
    • *kwargs：可选参数，用于传递其他参数，如请求头(headers)、超时(timeout)等。

  3. requests.put(url, data=None, **kwargs)

    发送PUT请求并获取响应。

    • url：请求的URL地址。
    • data：可选参数，字节序列，作为请求的正文内容。
    • *kwargs：可选参数，用于传递其他参数，如请求头(headers)、超时(timeout)等。

  4. requests.delete(url, **kwargs)

    发送DELETE请求并获取响应。

    • url：请求的URL地址。
    • *kwargs：可选参数，用于传递其他参数，如请求头(headers)、超时(timeout)等。

  5. requests.head(url, **kwargs)

    发送HEAD请求并获取响应的头部信息。

    • url：请求的URL地址。
    • *kwargs：可选参数，用于传递其他参数，如请求头(headers)、超时(timeout)等。

  6. requests.options(url, **kwargs)

    发送OPTIONS请求并获取响应。

    • url：请求的URL地址。
    • *kwargs：可选参数，用于传递其他参数，如请求头(headers)、超时(timeout)等。

  7. 其他常用函数：requests.patch()、requests.request()等。

这些函数提供了requests库的基本功能，可以根据具体的需求和情况进行扩展和定制。

使用案例：

这是一个简单的使用案例，演示了如何使用requests库发送GET请求并获取响应。

import requests

# 发送GET请求
url = "<https://api.example.com/data>"
response = requests.get(url)

# 获取响应内容
data = response.json()

# 处理响应数据
for item in data:
    print(item["name"], item["value"])

在这个案例中，我们使用requests.get()函数发送了一个GET请求，并将响应保存在response对象中。然后，我们可以使用response.json()方法获取响应的JSON内容，并对数据进行处理。在这里，我们假设响应是一个JSON数组，遍历数组并打印每个元素的名称和值。

请注意，在实际应用中，可能需要处理不同的HTTP方法、请求参数、请求头(headers)、超时(timeout)、身份验证等。requests库提供了丰富的功能和选项，可以根据具体的需求进行灵活的配置和使用。

代理的使用:

使用requests库发送请求时，可以通过设置proxies参数来指定代理服务器。以下是使用requests库发送请求时使用代理的示例：

import requests

# 定义代理服务器
proxies = {
    'http': '<http://proxy.example.com:8080>',
    'https': '<https://proxy.example.com:8080>'
}

# 发送请求并使用代理
url = "<https://api.example.com/data>"
response = requests.get(url, proxies=proxies)

# 处理响应
print(response.text)

在这个示例中，我们通过定义一个proxies字典来指定代理服务器的地址。字典的键是代理类型（http或https），值是代理服务器的URL。然后，我们在发送请求时，通过proxies参数将代理配置应用于请求。

请注意，代理服务器的URL格式应该符合代理服务器的要求。在示例中，我们使用了http://proxy.example.com:8080作为HTTP代理的URL，https://proxy.example.com:8080作为HTTPS代理的URL。你需要根据实际的代理服务器配置来设置正确的URL。

另外，如果代理服务器需要身份验证，你可以在代理URL中包含用户名和密码，例如：

proxies = {
    'http': '<http://username:password@proxy.example.com:8080>',
    'https': '<https://username:password@proxy.example.com:8080>'
}

在这个示例中，username是代理服务器的用户名，password是代理服务器的密码。

通过设置proxies参数，你可以在requests库中使用代理服务器发送请求，并在爬取数据时隐藏真实的IP地址和身份信息。
